# KDNA Trademark Policy

The KDNA name, the AIKDNA name, the KDNA logo, the Knowledge DNA mark, and related visual identity elements ("KDNA Marks") are trademarks of the KDNA project, maintained by the AIKDNA organization.

## What This Means

1. **KDNA is an open standard.** The protocol specification, file format, schemas, and validation rules are open for anyone to implement, use, and build upon.

2. **The KDNA Marks are not open for unrestricted use.** Code being open source does not mean the project identity and trademarks are open for any use.

3. **You may use the KDNA Marks to refer to KDNA.** You may say "compatible with KDNA," "supports KDNA," "KDNA domain," "KDNA tool," or "built for KDNA" when describing your implementation, tool, or domain package — provided the reference is truthful and not misleading.

4. **You may not use the KDNA Marks in ways that suggest endorsement, official status, or affiliation without permission.** This includes:
   - Naming your project "KDNA Something" in a way that implies it is an official KDNA project
   - Using the KDNA logo as your project's primary identity
   - Claiming "Official KDNA" or "Certified KDNA" status without authorization
   - Creating a fork and presenting it as the KDNA standard

5. **Compatible implementations must distinguish themselves.** If you build a tool, loader, validator, or platform that works with KDNA, name it clearly as your own project (e.g., "Acme KDNA Loader," not "KDNA Loader").

6. **Forks must be clearly identified as independent.** If you fork the KDNA repository, you must:
   - Clearly state that your fork is independent and not the official KDNA project
   - Remove or replace KDNA Marks from your fork's primary branding
   - Not redirect users to your fork as "the" KDNA standard

## Rationale

KDNA is built to become shared infrastructure for AI agent domain judgment. For shared infrastructure to remain coherent, the standard's identity must be protected from fragmentation and confusion. Open code does not mean open identity.
