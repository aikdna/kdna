# RFC-0013: Judgment Asset Lifecycle (Archived)

**Status:** Withdrawn — retained as an archive marker

This proposal is not part of the current KDNA Core specification and introduces
no required fields, container entries, runtime behavior, or compatibility
commitments. Earlier drafts mixed exploratory authoring concepts with examples
and implementation notes that are not appropriate as public normative guidance.
They have therefore been retired rather than maintained as a live RFC.

For current, public-facing guidance, use:

- [KDNA SPEC](../SPEC.md) for the canonical asset protocol.
- [Consumption runtime guide](../docs/consumption-runtime.md) for the
  supported selection, composition, projection, trace, evaluation, and review
  workflow.
- [Authoring guide](../docs/authoring-guide.md) for creating and validating an
  asset.
- [Human lock](./human-lock.md) and [load profiles](./load-profiles.md) for
  the currently documented governance and loading mechanisms.

Future lifecycle proposals must be published as self-contained public RFCs:
they must describe only publicly reproducible behavior, clearly identify their
status, and avoid treating authoring metadata as a KDNA Core runtime contract
until it is adopted by the specification.
