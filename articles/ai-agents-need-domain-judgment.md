# Why Judgment Needs Its Own Asset

Facts tell an AI system what is known. Judgment expresses choices: what matters,
which tradeoff to prefer, where a boundary lies, and when to stop or escalate.

That judgment may already live in a Prompt, Skill, Policy, Memory, workflow, or
ordinary document. KDNA does not claim those carriers are inferior. It provides
an independent asset when the judgment needs its own identity, version,
integrity, provenance, authorization, projection, replacement, and rollback.

A `.kdna` file does not contain intelligence and does not guarantee a better
answer. It can carry personal taste, organizational policy, professional
practice, AI-authored judgment, or any other bounded judgment whose provenance
is declared accurately. The format is neutral about whether the content is
correct or desirable.

The user or Host must select the file explicitly or approve an exact
attachment. Saving or discovering a file is not consent. Current facts, user
intent, law, safety, system rules, and Host permissions remain above asset
content, and active use must be visible and reversible.

That is the role of KDNA: not to make AI stronger, but to let selected judgment
be carried and governed across compatible AI systems without silent mutation.
