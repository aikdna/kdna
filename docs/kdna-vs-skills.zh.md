# KDNA 与 Skills

Skills 和 KDNA 都是 Agent 的能力扩展机制，但解决不同层面的问题。

## 核心区别

**Skills 解决：Agent 能做什么？**
**KDNA 解决：Agent 凭什么这样判断？**

Skill 是一个可重复的工作流。KDNA 是一个可加载的判断框架。

## 对比

| | Skills | KDNA |
|---|--------|------|
| 类型 | 执行包 | 判断包 |
| 内容 | 步骤、模板、工具调用、脚本 | 公理、概念边界、信号、误解、自检 |
| 触发 | 用户明确请求 | 领域匹配或关键词触发 |
| 加载 | 按需执行 | 进入领域前静默加载 |
| 输出 | 完成一个具体任务 | 改变 Agent 的判断方式 |
| 例子 | "生成周报" "格式化代码" | "价格异议是确定性不足" "讨论不是决策" |
| 可组合 | 可以串联 | 可以组合集群 |

## 它们一起工作

> Skills 执行流程。KDNA 塑造判断。两者互补，不互斥。

例子：一个销售 Agent。Skill 帮它调用 CRM、生成邮件、更新客户记录。KDNA 帮它判断客户的真实顾虑是预算不足还是确定性不足——这决定了整个后续流程的方向。

## 什么时候用哪个

| 场景 | 用 |
|------|-----|
| 用户说"帮我写封邮件" | Skills（有模板和流程） |
| 用户说"这封回复合适吗" | KDNA（需要领域判断） |
| 用户说"按这个格式生成报告" | Skills |
| 用户说"客户说太贵了" | KDNA（需要诊断信号） |
