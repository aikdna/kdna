# First KDNA Walkthrough — Local Demo Asset

This walkthrough uses a local packaged `.kdna` file generated by the CLI. It is
the same path a first-time user can run without downloading a separate example
asset.

When public example `.kdna` files are released, each one should have its own
release card with a download URL, SHA256 digest, boundaries, known limitations,
and before/after notes. Until then, this walkthrough proves the file-format and
loading path without relying on an unpublished asset.

---

## 1. Install the CLI

```bash
npm install -g @aikdna/kdna-cli
```

## 2. Generate a local `.kdna` asset

```bash
kdna demo minimal ./minimal
kdna pack ./minimal ./minimal.kdna
```

This creates a packaged `.kdna` file. The source folder is only the authoring
view; the user-facing asset is `minimal.kdna`.

## 3. Validate and plan loading

```bash
kdna validate ./minimal.kdna
kdna plan-load ./minimal.kdna
```

Expected validation result:

```json
{
  "format_valid": true,
  "schema_valid": true,
  "payload_valid": true,
  "checksums_valid": true,
  "load_contract_valid": true,
  "overall_valid": true,
  "problems": []
}
```

The LoadPlan should return `can_load_now: true` for this public local asset.

## 4. Load compact judgment context

```bash
kdna load ./minimal.kdna --profile=compact --as=prompt
```

The compact profile emits agent-readable judgment context. The minimal demo is
small by design; it proves the mechanics:

- the `.kdna` container can be opened safely;
- manifest, payload, checksums, and load contract validate;
- LoadPlan allows loading before judgment context is rendered;
- the compact profile can be passed to an agent workflow.

## 5. Inspect the full JSON

```bash
kdna load ./minimal.kdna --profile=full --as=json
```

Use the full profile for inspection, testing, and tool integration. Public
example assets should show richer judgment structures; this demo intentionally
stays small so the first-run path is easy to verify.

---

## Summary

| Step | Command | What it proves |
|---|---|---|
| Generate | `kdna demo minimal ./minimal` | The CLI can create a local source view |
| Package | `kdna pack ./minimal ./minimal.kdna` | The public asset is a single `.kdna` file |
| Validate | `kdna validate ./minimal.kdna` | Format, schema, payload, checksums, and load contract pass |
| Plan-load | `kdna plan-load ./minimal.kdna` | The asset is loadable before context is rendered |
| Compact load | `kdna load ./minimal.kdna --profile=compact --as=prompt` | Agent-ready judgment context exists |

A KDNA domain is not a prompt, a knowledge base, or a GitHub repository. It is a
portable `.kdna` judgment asset that an agent can validate, plan-load, and load.
