# 作用域错配与边界泄漏

兼容 Runtime 不能把判断资产投影到其声明作用域之外，也不能丢掉约束其使用的
边界。这里有两种不同缺陷：

- **作用域错配**：已经选择的资产不适用于当前任务；
- **边界泄漏**：资产相关，但投影缺失理解该判断所必需的排除项、例外或失败风险。

这是加载与投影缺陷。它不表示 KDNA 垄断所有判断，也不表示某套判断在脱离具名
评价者和作用域后可以被普遍称为“正确”或“错误”。

## 必要行为

在加载前，消费者应：

1. 检查身份、版本、兼容性和声明作用域；
2. 从兼容 Core 获取 LoadPlan；
3. 满足声明的授权状态；
4. 选择保留必要判断与边界的投影；
5. 无法保存最小语义单元时 fail closed；
6. 保证当前事实、用户指令、权限和安全政策高于资产内容。

当合同不能确认任务适用性时，Host 应跳过或询问，而不是自行假装确定。Host 可以
实现自己的适用性政策，但该政策不能被当成 Core 验证或通用质量判断。

问题记录应给出精确资产版本、任务、Host、LoadPlan、投影以及本应控制加载的边界，
并区分缺陷发生在创作、选择、授权、投影还是 Host 行为。
