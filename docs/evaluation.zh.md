# 评估一个 KDNA 资产

> [English](./evaluation.md)

评估属于提出主张的人或产品。它不是 KDNA Core 的格式有效性、官方质量分数，
也不是创建有效资产的前提。

必须分开以下事实：

1. **格式有效**：容器和 Schema 有效；
2. **完整性与来源**：字节、摘要和声明的来源相符；
3. **授权与交付**：精确资产经过允许、投影并交付给具名 Host；
4. **采用**：在具名任务和方法下，输出与所选判断一致；
5. **外部评价**：具名评价者按明确标准偏好、拒绝或对结果无差异；
6. **现实结果**：在明确情境中观察到了现实结果。

一层永远不能证明下一层。Core 尤其不能把外部评价变成资产内置的“优秀”、
“专业”、“安全”或“推荐”元数据。

任何公开主张都应给出主张者、精确资产版本与摘要、Host 与 Runtime 坐标、
作用域、方法、证据位置和已知限制。不同的人可以合理地对同一资产作出不同
判断。
