# 在 KDNA 之上构建商业服务

KDNA 是开放的判断资产协议。任何人都可以在它之上构建商业产品，但创建、发布、加载和使用合法 KDNA 资产，不依赖任何商业服务。

## 开放协议

公开规范、Schema、Core SDK、CLI、创作工具、Loader 和 conformance fixtures 构成共同基础。格式是否合法，与作者是谁、是否收费无关。

## 作者控制的资产

作者可以在适用法律和自己选择的许可范围内发布资产。KDNA Core 不认证作者是否专业，不判断内容质量，也不规定商业模式。

## 三种访问方式

| 访问方式 | 资产边界 | 可能用途 |
|---|---|---|
| `public` | 内容不保密；Agent 消费仍通过 Capsule | 开放参考资产、社区发布、免费作品 |
| `licensed` | 判断载荷加密；授权后只在内存中投射 | 付费资产、会员内容、组织内分发 |
| `remote` | 完整判断载荷留在作者或部署者服务器 | 托管任务投射、敏感组织判断 |

访问方式由作者选择。工具链执行声明的访问和授权事实，但不判断资产内容。

## 生态可以构建什么

独立创作者和公司可以提供创作服务、托管、分发、发现、评测、集成、授权、支持或垂直应用。这些都是建立在 KDNA 之上的产品，不是 KDNA Core 的必需组成，也不代表 AIKDNA 为其判断内容背书。

## 当前公开状态

当前公开基线包括协议、本地工具链、自主发布和可自托管参考组件。AIKDNA 托管的 marketplace、billing、registry、认证和 remote loading 目前不是公开服务。生产使用 licensed 或 remote 流程前，应检查对应包的 release notes。
