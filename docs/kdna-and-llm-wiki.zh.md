# KDNA 与 LLM Wiki

从原始材料到 Agent 判断力的实践管线。

## 问题

大型语言模型可以处理维基百科、文档库、知识库。但原始材料不等于判断力。

维基百科可以告诉你"什么是销售"。但它不会告诉你"客户说太贵了的时候，先检查确定性还是先检查预算"。

## KDNA 的思考管线

```
LLM Wiki / 文档库 / 专家经验
        ↓
   提取核心判断：什么信号重要？什么常被误判？什么条件必须满足？
        ↓
   编码为 KDNA：公理、本体、框架、误解、自检
        ↓
   Agent 加载 → 判断力形成
```

KDNA 不是 LLM Wiki 的替代品。它是 LLM Wiki 的**上一层**——把原始知识转化为判断结构。

## KDNA for LLM Wiki 提示词

以下是一个可复用的提示词模板，用于从原始材料中提取 KDNA：

```
你是一位领域判断架构师。从以下原始材料中，提取出该领域专家在面对具体问题时的判断结构。

请按照以下结构输出：

1. 公理（2-4条）
   - 每条公理必须是一个可验证的判断原则，不能是泛泛的建议
   - 格式：一句话 + 完整解释 + 为什么这条公理对 Agent 的判断至关重要

2. 本体概念（2-4个）
   - 定义该领域中的核心概念
   - 每个概念必须有明确的边界（它不是什么，常与什么混淆）
   - 必须有可观测的触发信号

3. 禁用词（3-5个）
   - 列出该领域中容易误导 Agent 的词汇
   - 每条说明为什么误导，以及该用什么替代

4. 常见误解（2-4个）
   - Agent 在没有领域判断时可能产生的错误判断
   - 每条对比错误理解和正确理解

5. 自检问题（3-5个）
   - Agent 在输出前必须自问的 yes/no 问题
   - 必须是该领域特有的，不能是通用的"回答是否有帮助？"

原始材料：
---
[粘贴 LLM Wiki 内容或专家文档]
---
```

这个提示词是 `kdna-create` Skill 的核心逻辑。
