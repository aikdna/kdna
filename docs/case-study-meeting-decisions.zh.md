# 案例研究：四万美元的会议

一个复合案例，展示 KDNA 如何防止 Agent 将讨论误判为决策。

## 场景

一家 SaaS 公司的高管层开了一次产品方向会议。会上讨论了三个话题：

1. 是否进入企业市场
2. 当前产品架构是否需要重构
3. 下个季度的招聘优先级

会议持续了 90 分钟。有人提出想法，有人表达顾虑，有人表示支持。会议结束时，大家感觉"方向明确了"。

## 传统总结

一个没有 KDNA 的 Agent 收到会议记录后，会生成如下总结：

> 会议达成以下决策：
> 1. 启动企业市场探索
> 2. 重构认证模块
> 3. Q3 优先招聘后端工程师

这看起来高效、专业——但它是虚假的。

## KDNA 的判断

加载 `decision_state` KDNA 后，Agent 逐一检查每个"决策"：

**话题 1：企业市场**

- 是否有人被指定负责？→ 否
- 是否有明确的时间节点？→ 否
- 是否有成功标准？→ 否
- 是否有人提出质疑但被搁置？→ 是（CFO 担心成本，但"后续再议"）

判断：**UNRESOLVED** — 有讨论，无决策。

**话题 2：架构重构**

- 是否指定了负责人？→ 是（CTO）
- 是否有时间节点？→ "Q3 前完成评估"
- 是否有明确范围？→ 有条件限制（"如果新方案性能达标就采用"）

判断：**CONDITIONAL** — 有条件决策。不应该被当作确定的行动计划。

**话题 3：招聘优先级**

- 是否指定了负责人？→ 是（HR 负责人）
- 是否有时间节点？→ 是（本周发 JD）
- 是否有明确选择？→ 是（后端优先于前端）

判断：**EXECUTABLE_DECISION** — 这个才是真正可执行的决策。

## 成本

如果 Agent 把三个"决策"都当作确定的行动计划：

- 企业市场"决策"会产生大量准备工作和支出，但没有人真正负责推动
- 架构重构的"条件"会被忽略，团队开始执行一个还没有被验证可行的方案
- 招聘决策虽然正确，但与其他虚假决策混在一起，失去了优先级

估算浪费：大约 4 万美元的人力成本和至少一个季度的方向混乱。

## KDNA 的作用

KDNA 不改变会议的内容。它改变的是**Agent 对会议内容的理解方式**。

通过加载决策判断领域的公理和框架，Agent 学会：
- 区分讨论和决策
- 识别缺失的决策要素（负责人、时间、条件）
- 不在虚假的确定性上生成行动计划

这就是 KDNA 的核心价值：同样的输入，不同的判断。
