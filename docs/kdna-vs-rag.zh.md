# KDNA 与 RAG

RAG（检索增强生成）和 KDNA 都在 Agent 行动前提供上下文——但提供的东西完全不同。

## 核心区别

**RAG 回答：关于这件事，有什么相关信息？**
**KDNA 回答：这件事在特定领域里，意味着什么？**

RAG 给你事实。KDNA 给你判断标准。

## 对比

| | RAG | KDNA |
|---|-----|------|
| 数据源 | 文档、数据库、网页 | 专家判断结构 |
| 内容 | 不相关的原文段落 | 公理、概念、框架、自检 |
| 更新 | 重新索引 | 版本化发布 |
| 时效性 | 实时检索 | 按需更新 |
| 结构 | 非结构化文本块 | 结构化 JSON |
| 例子 | "查一下公司定价政策" | "价格异议意味着确定性不足" |

## 它们一起工作

> RAG 检索"客户上次购买是什么时候"。KDNA 判断"频繁询问价格意味着什么"。

两者不冲突。RAG 提供信息原料，KDNA 提供判断框架来解释这些原料。

最好的 Agent 同时使用两者：用 RAG 获取上下文，用 KDNA 在上下文中做领域判断。

## 什么时候用哪个

| 场景 | 用 |
|------|-----|
| "公司的退货政策是什么" | RAG |
| "这个客户为什么反复退货" | KDNA（需要诊断信号模式） |
| "查找上次会议记录" | RAG |
| "这次会议达成了什么决策" | KDNA（需要判断决策状态） |
