# Studio 创作原则

KDNA Studio 帮助个人、Agent、工具或混合流程把判断显性化，复核其边界，并导出
带版本的 `.kdna` 文件。它是创作环境，不是判断内容真伪或优劣的权威。

## 作者中立

有效的 KDNA 资产可以由人、AI 系统、组织、工具或混合流程创建。创作流程必须
准确声明来源。

只有当导出声称代表某个具名个人或机构时，才必须由该主体确认被代表的判断。
如果没有这种代表性声明，主体确认不是 Core 有效性的条件。

## 最小流程

1. 明确判断和预期作用域；
2. 表达有实际选择意义的取舍、边界和失败风险；
3. 复核并修订内容；
4. 记录准确来源以及任何被声明的主体确认；
5. 导出一个带版本的 `.kdna` 文件；
6. 通过 Core 对该精确文件执行验证、计划和加载。

AI 可以协助访谈、结构化、挑战或编译，但不能伪造人工确认、隐藏不确定来源，
也不能把外部评价变成格式有效性的门槛。

## 可选作者验收

作者可以用少量未参与创作的情境，检查资产是否表达了预期方向、遵守边界，
并在当前事实、用户意图、法律、安全和 Host 权限面前正确让位。这是作者验收，
不是智力提升或优越性测试。

## Studio 边界

Studio 负责创建、编辑、来源记录和导出。它不负责：

- 通用内容审批或排名；
- 强制 Registry 或市场；
- Agent 权限或自主选择资产；
- 组织级完整治理基础设施；
- 保证使用资产一定带来更受偏好的结果。

实验性创作模块在其产品需求和合同获得确认前，必须退出默认公共导出、CLI 与
发布包表面。
