# AI Agent 缺乏判断力

架构文章：为什么 Agent 生态解决了动作，但没有解决辨别力。

## 当前 Agent 栈

现代 AI Agent 的能力已经很全面了：

| 层 | 作用 | 例子 |
|----|------|------|
| 大模型 | 推理和生成 | GPT、Claude |
| 工具/MCP | 外部操作 | API 调用、文件系统、数据库 |
| RAG | 知识检索 | 文档问答、语义搜索 |
| 记忆 | 历史和连续性 | 会话历史、用户偏好 |
| Skills | 可重复工作流 | 模板、脚本、自动化 |

## 缺少的一层

所有这些层解决了一个共同的问题：**Agent 能做什么？**

但它们没有解决另一个问题：**Agent 凭什么判断得对？**

一个 Agent 可以调用 CRM、检索客户历史、生成邮件——但如果它不理解"客户第三次问价格"意味着什么，所有能力都在错误的方向上运行。

这不是 Agent 能力的问题。这是 Agent 缺少领域判断框架的问题。

## 症状

以下症状说明 Agent 缺少判断层：

- 给用户太多信息，不知道什么更重要
- 能生成多个合理的方案，但不知道哪个更可靠
- 能执行任务，但不知道什么时候该停止、升级或拒绝
- 能模仿专家语言，但没有专家的判断结构
- 把讨论当成决策，把噪音当成信号，把偏好当成约束

## KDNA 补全这个栈

KDNA 不替代底层任何一层。它增加了一个新的层：

```
大模型       → 推理
工具/MCP     → 行动
RAG         → 知识
记忆        → 历史
Skills      → 流程
KDNA        → 判断  ← 这是缺失的
```

KDNA 告诉 Agent：在这个领域里，该注意什么、该避开什么、什么信号意味着什么、行动前必须确认什么。

这不是给 Agent 更多信息。这是给 Agent 一个判断框架。

## 结论

Agent 生态在过去两年解决了"Agent 能做什么"。下一个阶段需要解决"Agent 是否判断得对"。

KDNA 就是为下一个阶段准备的基础设施。
