# 国际化指南

KDNA 如何支持多语言领域。

## KDNA 的语言模型

KDNA 领域包可以声明一种或多种支持语言：

```json
{
  "language": "zh",
  "languages": ["zh", "en"]
}
```

- `language`：领域包的主要语言
- `languages`：所有支持的语言列表

## 创建多语言 KDNA

目前 KDNA 支持两种多语言策略：

### 策略 1：每种语言一个独立的 KDNA 包

```
writing-kdna/        ← 英文版
writing-kdna-zh/     ← 中文版
```

两个独立的包，各自独立维护和版本化。

**优点**：每种语言可以有自己的公理和概念（因为判断结构在不同语言中可能不同）。
**缺点**：维护成本翻倍。

### 策略 2：单包多语言字段

在同一个 JSON 中提供多语言内容：

```json
{
  "axioms": [{
    "id": "AX-001",
    "one_sentence": "Clarity is the writer's only obligation.",
    "one_sentence_zh": "清晰是写作者唯一的义务。",
    "full_statement": "...",
    "full_statement_zh": "..."
  }]
}
```

**优点**：一个包，双语覆盖。
**缺点**：JSON 文件变大，字段名需要约定。

## 当前推荐

当前阶段推荐**策略 1**（每种语言独立包）。原因：
- 不同语言的判断结构确实可能不同
- 版本管理更清晰
- Agent 根据用户语言自动选择对应包

未来可能支持策略 2 作为可选的便利方案。

## 中文 KDNA 创作注意事项

中文 KDNA 的编写有一些特殊考虑：

1. **触发信号**必须是中文真实语境中出现的表达——不能直接翻译英文信号
2. **禁用词**必须是中文语境中常见的误导性表达
3. **公理**需要反映中文思维方式的判断差异

详见 `kdna-in-chinese.md`。
