# 为什么叫 KDNA

KDNA 的名字来自 DNA 的启发：**如果 DNA 是生命体可继承、可表达、可变异、可进化的底层编码，那么 KDNA 希望成为 AI Agent 时代"领域判断力"的底层编码。**

K 代表 **Knowledge**，DNA 代表一种可复制、可组合、可进化的结构。

> Knowledge DNA：让专业判断力像 DNA 一样被结构化、被继承、被更新、被传播，并被 AI Agent 调用。

## DNA 带给我们的核心启发

DNA 最打动人的不是它储存了生命信息，而是它具备四个特征：

1. **不是零散信息，是有结构的编码。** 生命不靠散乱片段运转，而是靠稳定、可读取、可表达的遗传结构。

2. **可以被复制和继承。** 一个生命体的核心信息，可以传递给下一代，并在新环境中继续表达。

3. **可以发生变异和进化。** DNA 不是静止的档案，而是在复制、选择、适应中持续演化。

4. **连接了微观结构和宏观表现。** 很小的编码差异，最终可能影响一个生命体的形态、能力和适应方式。

这正是专业能力的本质：真正有价值的往往不是表面知识，而是背后的"判断结构"。

一个真正优秀的医生，不只是记住医学知识，而是知道哪些信号意味着高风险。
一个真正优秀的管理者，不只是懂管理术语，而是知道什么时候该授权、什么时候该干预。
一个真正优秀的写作者，不只是会写句子，而是知道什么表达有力量、什么表达会误导读者。

这些能力，就像"领域判断力的 DNA"。

## 为什么不是 Knowledge Base

知识库回答：**有哪些知识？**

KDNA 回答：**面对一个具体问题，应该如何判断？**

知识库像资料室。KDNA 像专家的大脑结构。

知识库储存答案、文档、案例、术语。
KDNA 描述标准、边界、取舍、风险、信号、优先级和判断路径。

所以不叫 KBase、Prompt Pack 或 Agent Rules——那些名字太像"材料"或"指令"。KDNA 想表达的是更底层的东西：一套可被 AI Agent 继承和表达的领域认知结构。

## KDNA 想成为什么

KDNA 不是一个单一应用。它是一个开放标准和基础设施。

长期愿景：

> 让人类的领域判断力，可以像代码一样被创建、验证、版本化、开源、安装、组合、授权、交易，并被不同 AI Agent 调用。

KDNA 官方不生产所有领域的 KDNA。更好的生态是：

- **KDNA 官方**：定义标准、提供 CLI、验证工具、加载工具和协议
- **开发者 / 专家 / 企业**：创建自己的领域 KDNA
- **GitHub**：承载开源协作与版本更新
- **未来 Registry**：承载安装、发现、授权和交易
- **Agent**：消费这些领域判断力

KDNA 的真正使命不是成为所有领域知识的仓库，而是成为：**AI 时代领域判断力包的开放标准。**
