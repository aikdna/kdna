# 为什么叫 KDNA

KDNA 是 **Knowledge DNA** 的比喻：被选择的判断可以编码为结构化形式，在不被
静默改写的前提下复制，用版本和摘要识别，并在多个兼容 Host 中表达。

这个比喻不意味着 KDNA 包含智力、人格、真理或完整心智，也不意味着它让模型
变强。KDNA 给一套有边界的判断提供独立资产和加载合同。

## 为什么是判断，而不只是知识

文档和知识库通常保存事实与解释。判断还包含选择：什么更重要、偏好哪种取舍、
边界在哪里、什么风险需要升级，以及规则何时不适用。

Prompt、Skill、Policy、Memory、工作流和普通文档都可以携带同样的判断。
当这套判断需要脱离某一个载体获得身份和生命周期——版本、完整性、来源、授权、
投影、替换与回滚——KDNA 才增加价值。

## 什么可以演进

作者可以持续修订判断资产。每次修订都必须成为可识别的新版本；Host 不能静默
替换已经附加的版本。演进是作者或组织的选择，不代表新判断自动更好。

## 使命

KDNA 提供开放格式和工具链，让个人与组织可以创建、携带、检查、授权并在兼容
AI 系统之间使用自己的有边界判断资产。
