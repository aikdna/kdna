# KDNA 与 AI 技术栈

## KDNA 和知识库、记忆、Skill、MCP、Workflow、Prompt 到底是什么关系？

KDNA 不替代已有的 AI 技术栈。不替代知识库、记忆、Skill、MCP、Workflow、Prompt、RAG、评估体系、微调。

KDNA 之所以存在，是因为这些组件常常需要一个清晰的判断参考。

一句话：

> **KDNA 不是又一个工具层。KDNA 是判断层，用来指导其他层如何被使用。**

---

## 1. 核心边界

AI 技术栈包含很多有用的层级：

| 组件 | 主要角色 | 给 Agent 什么 |
|------|---------|-------------|
| **LLM** | 通用智能 | 语言、推理、生成 |
| **Prompt** | 任务指令 | 现在做什么 |
| **知识库** | 信息来源 | 事实、文档、参考资料 |
| **记忆** | 上下文连续性 | 过去的交互、偏好、历史 |
| **Skill** | 过程 | 如何执行一个可重复的任务 |
| **MCP** | 工具和数据连接 | 标准化访问外部系统 |
| **Workflow** | 执行结构 | 步骤、依赖、任务顺序 |
| **RAG** | 检索 | 外部上下文和文档 |
| **Eval** | 测量 | 输出是否通过测试 |
| **微调** | 模型行为塑造 | 内化到模型权重中的模式 |
| **KDNA** | **判断系统** | **什么重要、什么该拒绝、什么算好** |

KDNA 不和这些组件竞争。KDNA 给它们提供判断参考。

> 知识告诉 Agent 有什么。记忆告诉 Agent 发生过什么。Skill 告诉 Agent 怎么做。MCP 告诉 Agent 能连什么。Workflow 告诉 Agent 按什么步骤。KDNA 告诉 Agent 如何判断。

---

## 2. KDNA vs 知识库

知识库存储信息：文档、笔记、文章、手册、政策、案例、FAQ、研究资料、结构化数据。

知识库回答：*有什么信息？文档怎么说？什么事实相关？有什么案例可以参考？*

KDNA 回答不同的问题：*信息该如何被解释？什么信息最重要？什么区分不能模糊？哪个来源需要谨慎对待？什么风险应该阻断行动？什么标准决定答案好坏？*

知识库给 Agent 材料。KDNA 给 Agent 判断标准，告诉它如何使用材料。

**例子：** 企业知识库可能包含所有历史客服工单。客服 KDNA 领域则编码了如何判断紧急程度、何时升级、什么语言保持信任、什么风险需要人工审核、什么模式暗示法律或安全问题。

> 知识库告诉发生了什么。KDNA 告诉如何判断什么重要。

---

## 3. KDNA vs 记忆

记忆随时间保留上下文：用户偏好、历史对话、个人事实、项目历史、重复指令、过往决策。

记忆回答：*之前发生过什么？这个用户通常偏好什么？已经决定了什么？*

KDNA 回答：*Agent 应该如何判断这类情境？哪些原则在跨会话中始终适用？哪些边界必须保持？哪些自检必须通过？*

记忆是上下文性的。KDNA 是领域判断性的。两者配合使用。

**例子：** 记忆可能记住用户偏好简洁写作。写作 KDNA 领域则判断草稿是否有真正论点、是否证据充足、是否有受众张力、结构是否支撑论点。

> 记忆个性化。KDNA 做判断。

---

## 4. KDNA vs Skill

Skill 封装可重复的能力或过程：转录音频、生成字幕、运行测试、编辑视频、调用 API、转换文件、创建图表、部署软件。

Skill 回答：*怎么执行这个任务？应该调用什么工具？应该产出什么文件？*

KDNA 回答：*这个情境是否应该使用这个过程？结果应该达到什么质量标准？应该避免什么失败模式？什么时候输出不够好？什么时候应该停下来问人？*

Skill 执行。KDNA 判断执行。

**例子：** 字幕 Skill 可以提取音频、转写语音、对齐时间戳、导出 `.srt` 文件。字幕质量 KDNA 领域则定义什么是可读的字幕分段、如何检测识别错误、什么时机偏移不可接受、什么时候需要人工审核。

> Skill 告诉 Agent 怎么做。KDNA 告诉 Agent 如何判断该不该做、做得怎么样、什么时候不该继续。

---

## 5. KDNA vs MCP

MCP 通过标准接口将 AI 系统连接到工具、数据源和外部服务。

MCP 回答：*有什么工具可用？怎么调用这个系统？能访问什么数据？能执行什么操作？*

KDNA 回答：*什么时候该用这个工具？使用它有什么风险？哪个结果应该信任？什么操作需要确认？什么领域标准应该约束工具使用？*

MCP 提供访问。KDNA 提供关于访问和使用的判断。

**例子：** MCP 服务器可能让 Agent 访问 CRM 系统。销售 KDNA 领域则定义什么线索应该被视为高意向、哪些客户信号是误导、什么承诺不该做、什么时候需要真人销售接手。

> MCP 连接 CRM。KDNA 引导如何判断 CRM 数据。

---

## 6. KDNA vs Workflow

Workflow 定义执行顺序：步骤、依赖、分支逻辑、审批、重试、交接、自动化、状态转换。

Workflow 回答：*先做什么？再做什么？哪一步依赖哪一步的输出？什么时候该推进流程？*

KDNA 回答：*这个情境用这个流程对吗？每一步应该由什么判断来约束？什么风险应该中断流程？什么输出不够好不能继续？什么异常需要升级？*

Workflow 组织行动。KDNA 判断行动。

**例子：** 内容发布 Workflow 可能包含选题、提纲生成、草稿撰写、编辑、配图、排期、发布。内容策略 KDNA 领域则判断选题是否有真正的受众张力、标题是否制造了虚假紧迫感、草稿是否有真正洞察、这篇内容是否应该发布。

> Workflow 推进流程。KDNA 决定推进是否合理。

---

## 7. KDNA vs Prompt

Prompt 在当前上下文中给 AI 一条指令。Prompt 回答：*AI 现在该做什么？*

KDNA 回答：*什么领域判断应该跨多个任务持续指导 Agent？什么区分必须在会话间保持？什么错误应该反复避免？什么原则定义了这个领域的好工作？*

Prompt 通常是任务级的。KDNA 是领域级的。

**例子：**
- Prompt：*"帮我把这篇文章改好"*
- KDNA：*"不要先把写作当成语言润色问题。先诊断文章是否有真正论点、受众张力、足够证据和支撑结构，再编辑文字。"*

> Prompt 要求一个动作。KDNA 定义动作背后的判断。

---

## 8. KDNA vs RAG

RAG 从外部源检索相关信息并提供给模型。

RAG 回答：*应该检索什么上下文？哪些文档相关？什么段落支持答案？*

KDNA 回答：*检索到的信息该如何被判断？什么才算是充分证据？哪些检索材料只该当背景？什么区分应该引导解读？什么风险应该阻止过度自信使用？*

RAG 把信息带进上下文。KDNA 告诉 Agent 如何评估和应用这些信息。

**例子：** 医疗 RAG 系统可能检索临床文档。医疗风险 KDNA 领域则定义不确定性处理、升级边界、禁止的过度断言、以及何时 Agent 必须告诉用户咨询专业人士。

> RAG 检索。KDNA 约束判断。

---

## 9. KDNA vs Eval

评估体系衡量 AI 系统表现如何：测试用例、基准、评分标准、预期输出、通过/失败检查、人工审核标准。

Eval 回答：*输出通过了吗？结果有多好？什么行为改善或失败了？*

KDNA 回答：*什么判断系统应该在输出之前引导 Agent？什么原则应该被触发？什么误解应该避免？什么边界应该阻断输出？*

Eval 测量行为。KDNA 引导行为。

一个好的 KDNA 领域应该有 evals。Evals 可以测试 KDNA 是否按预期方向改变了 Agent 的判断。这就是 Judgment Delta 的重要性：相同输入、相同模型、不加 KDNA vs 加 KDNA。

---

## 10. KDNA vs 微调

微调通过训练模式到模型权重中来改变模型行为。

微调回答：*怎么让模型内化一种行为模式？怎么在模型层面塑造行为？*

KDNA 回答：*怎么让选定的判断保持显性？人类怎么检查和编辑它？同一个判断怎么跨模型使用？判断怎么版本化、授权、组合和撤销？*

微调内化。KDNA 外化。微调可以很强大，但产生的结果可能更难检查、归因、精确编辑或跨模型迁移。KDNA 保持判断可见。

成熟系统可以两样都用：微调改善通用行为，KDNA 提供运行时的显式领域判断。

---

## 11. 常见误解

**"KDNA 不就是更好的 prompt 吗？"** 不是。Prompt 是任务级的指令。KDNA 是结构化的、领域级的判断系统，可以验证、版本化、组合、分发和复用。

**"KDNA 是知识库吗？"** 不是。知识库存储信息。KDNA 编码如何判断、排序、拒绝和应用信息。

**"KDNA 是记忆吗？"** 不是。记忆保留过往上下文。KDNA 提供领域判断标准。

**"KDNA 是 Skill 吗？"** 不是。Skill 执行过程。KDNA 判断过程和结果。

**"KDNA 是 MCP 吗？"** 不是。MCP 连接工具和数据。KDNA 指导何时以及如何使用这些工具和数据。

**"KDNA 是 Workflow 逻辑吗？"** 不是。Workflow 组织步骤。KDNA 定义应该约束这些步骤的判断。

**"KDNA 是微调的文件形式吗？"** 不是。微调在模型权重内修改行为。KDNA 保持领域判断显性、可携带、可检查、可编辑。

---

## 12. 最终原则

KDNA 不是要替代 AI 技术栈。KDNA 是要让判断层显性化。

未来的 AI 技术栈需要模型、Prompt、记忆、知识库、Skill、工具、Workflow、评估体系、微调。但当 Agent 越来越强大、越来越主动，它们也需要一种标准方式来加载人主导的领域判断系统。

这就是 KDNA 的角色。

> 知识告诉 Agent 有什么。记忆告诉 Agent 发生过什么。Skill 告诉 Agent 怎么做。MCP 告诉 Agent 能连什么。Workflow 告诉 Agent 按什么步骤。Eval 告诉 Agent 是否通过。**KDNA 告诉 Agent 如何判断。**
