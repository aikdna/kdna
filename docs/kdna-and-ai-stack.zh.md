# KDNA 与 AI 技术栈

## 边界是合同，不是谁垄断判断

判断本来就存在于整个 AI 技术栈中：模型行为、Prompt、Skill、Policy、Workflow、
Memory、知识库、检索文档、测试以及个人和组织经验都可以携带判断。

KDNA 不发明判断，也不声称其他载体没有判断。KDNA 为一套被选择、有作用域的
判断系统提供独立 `.kdna` 资产身份，以及完整性、授权、加载、投影和生命周期的
共享合同。

同一判断可以同时存在于 Prompt、Skill、Policy、文档和 `.kdna` 中，甚至在一个
具体任务中产生相同行为。这完全正常。KDNA 的额外价值是协议可移植性和资产
控制，不是天然更聪明、判断更好或行为更强。

## 主要责任可以重叠

| 系统 | 主要合同 | 可以携带判断吗 | KDNA 不替代什么 |
|---|---|---:|---|
| 模型 | 学习到的推理与生成 | 可以 | 模型能力和权重 |
| Prompt | 单次调用的指令与上下文 | 可以 | Prompt 工程和评测 |
| Skill | 可复用能力、资料和工具流程 | 可以 | Skill 安装和执行合同 |
| 知识库 / Wiki | 内容组织与访问 | 可以 | 文档真源与协作 |
| RAG / 搜索 | 查询时检索 | 可以 | 检索、引用和当前事实查询 |
| Memory | 持久化与连续性 | 可以 | 用户历史和应用状态 |
| Policy | 规则、原则、裁量和治理权威 | 可以 | 强制力和权限 |
| Workflow | 步骤、状态、审批和重试 | 可以 | 流程执行 |
| MCP / 工具 | 服务器、资源、Prompt 和工具交互 | 可以 | 能力访问和行动授权 |
| Evaluation | 按特定方法或 rubric 测量 | 可以 | 内容排名和结果证明 |
| KDNA | 判断资产身份与兼容加载 | 可以，且是 Payload 的主要责任 | 以上任何系统 |

“主要”不代表“唯一”。Skill 可以做情境分类，Policy 可以包含裁量判断，RAG
可以检索 rubric，Prompt 可以结构化并进入回归测试。反过来，`.kdna` 也可以
包含表达判断所需的事实前提、示例、方法说明和引用；Core 不会因此把这些前提
认证成世界真相，也不会把方法说明变成工具执行权。

## KDNA 与 Prompt

Prompt 可以结构化、版本化、测试、组合和检查。如果一套 Prompt 系统已经提供
所需的身份、部署和控制，就未必需要 KDNA。

当判断需要脱离某个 Prompt 实现获得独立生命周期，或者多个兼容 Runtime 必须
核验准确字节、授权、投影、替换或回滚时，KDNA 才增加价值。

## KDNA 与 Skill

Skill 可以封装指令、资料、脚本和工具流程，也可以包含大量判断。`.kdna` 并不
以“Skill 只有执行”为前提；它的主要合同是把判断作为资产分发和加载。

Skill 可以加载 KDNA，也可以直接内嵌相同判断，或者两者并用。只有当判断需要
独立于 Skill 更新、授权或复用时，分离资产才有额外价值。

## KDNA 与知识、RAG 和 Memory

知识系统和 Memory 可以保存事实、解释、偏好、规则和判断；RAG 可以检索这些
内容。它们的主要合同是存储、连续性和检索。

KDNA 不替代它们，也不是世界事实真源。KDNA 可以证明某个具名判断资产版本按
合同被检查、授权和投影；检索系统则可以提供资产不应复制的当前证据和来源材料。

## KDNA 与 Policy、Workflow、MCP 和工具

Policy 和 Workflow 经常包含判断。工具选择和代码执行同样需要安全、证据、停止
条件和授权方面的判断。

加载 KDNA 永远不授予行动权。权限来自用户、Host、Policy、身份与外部系统。
KDNA 可以为这些系统提供判断输入，但不能仅凭格式有效就执行规则、批准部署或
调用工具。

## KDNA 与 Evaluation

KDNA Core 验证格式和加载合同。Evaluator 可以按具名方法评价资产内容或观察到
的行为，这些结论属于评价者，不能变成 Core 内置的质量、风险、信任或推荐字段。

同一内容在 Prompt 与 `.kdna` 中产生相同行为，并不削弱 KDNA 的意义：行为影响
来自判断内容，KDNA 的独立主张是资产身份与交付合同。

## 什么时候使用 KDNA

存在多项下列需求时再使用 KDNA：

- 判断需要独立于某个应用或模型获得身份；
- 准确版本和字节完整性很重要；
- 必须检查授权或按任务投影；
- 同一资产需要跨兼容 Agent 或 Runtime 移动；
- 替换、撤销、回滚或消费证据很重要。

当 Prompt、Skill、文档、Policy、Memory 或检索系统自己的合同已经足够，就使用
更简单的机制。不是每一种判断都需要 `.kdna`。

## 一个任务中的重叠载体

文章发布系统可以同时使用：

| 组件 | 示例贡献 |
|---|---|
| Prompt | “把这篇草稿修订到可发布。” |
| 知识 / RAG | 品牌资料、受众研究和引用来源 |
| Memory | 用户当前偏好和过去决定 |
| Skill | 分析结构、编辑段落并准备 CMS 数据 |
| Policy / Workflow | 审批、法务复核和发布权限 |
| KDNA | 在多个编辑器之间共享的具名写作判断资产 |
| Evaluation | 按特定方法审查草稿或最终结果 |

每一行都可能含有判断。架构仍然清楚，因为各系统保留自己的权威和合同。

## 最终原则

> KDNA 是判断资产的开放格式与加载协议。它不拥有判断；它让一套被选择的判断
> 系统在明确技术合同下变得可携带。
