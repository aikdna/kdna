# 为什么需要 KDNA

大模型已经很强了。它们会写、会搜、会总结、会调用工具。Agent 越来越像能自主行动。

但当 Agent 变得越来越强之后，一个更关键的问题出现了：

> 当 AI 已经会行动之后，它凭什么判断得对？

## 问题不在信息量，在判断结构

AI 不缺少信息。它可以检索、可以读取文件、可以调用 API。

缺少的是：**面对一个具体场景，该先看什么？什么是信号？什么是噪音？什么条件不满足就不能行动？**

这个问题，Skill 解决不了。Skill 告诉 Agent **做什么**。它不告诉 Agent **怎么判断**。

这个问题，RAG 也解决不了。RAG 告诉 Agent **有什么知识**。它不告诉 Agent **知识怎么用**。

## 一个具体例子

团队开了个会。会上讨论了技术债务，有人点头，有人说"回头再看看"。

普通 AI 总结："团队讨论了技术债务，计划收集数据后再决定。"

KDNA 加载后判断：**这不是一个决策。** 没有明确选择、没有指定负责人、没有时间节点、没有成功标准。这是一个 UNRESOLVED 状态。

这个判断差异至关重要。如果把讨论当成决策，Agent 会生成行动项，团队会以为事情在推进——实际上没人负责。

## KDNA 解决的根本问题

| 层 | 做什么 | 例子 |
|----|--------|------|
| Prompt | 控制表达 | "用友好的语气" |
| Skill | 执行流程 | "按这个模板生成周报" |
| MCP | 连接工具 | "调用 Jira API" |
| RAG | 检索知识 | "查找公司定价政策" |
| **KDNA** | **塑造判断** | **"价格异议是确定性不足，不是价格问题"** |

KDNA 不替代任何一层。它填补了所有层之间的空白：**在行动之前，先判断。**

## 为什么不是一次性的

领域判断力不是提示词。提示词是一次性的指令。KDNA 是可以被版本化、验证、安装、更新的结构。

一个写作 KDNA 可以从 v0.1 开始，到 v0.2 增加新能力，到 v1.0 形成稳定的判断框架。

这意味着 KDNA 不只是让今天的 Agent 判断得更好——它让判断力本身可以积累和进化。
