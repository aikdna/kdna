# Cross-Agent Smoke Test — Withdrawn Baseline

> **Status:** Historical. This file is not a current interoperability claim or
> release gate.

The former matrix treated Skill installation, global asset discovery, and an
expected answer change as if they proved correct KDNA use. They do not.

A future cross-Host acceptance run must start from one explicitly selected
`.kdna` file or an exact user-approved attachment. For each Host it must record:

- the exact asset version and digest;
- `validate`, LoadPlan, and Runtime Capsule results;
- the user-approved attachment scope;
- a visible active-asset indicator and disable/switch path;
- whether judgment use was observed, separately from external outcome;
- the exact Host and adapter coordinates.

Codex, Claude Code, and OpenCode must consume the same frozen asset through
their own Host adapters. Installing a Markdown Skill or finding a file on disk
is not acceptance evidence.
