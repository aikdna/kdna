# Agent 适配器行为边界

> 当前合同方向。仓库中的 `kdna-loader` 在实现和安装流程通过本边界复验前，仍是
> Unassessed，不能作为发布权威。

## 适配器不是权威

Agent 适配器只能加载：

1. 用户为当前任务明确提供路径的 `.kdna`；或
2. Host 已批准用于当前工作区、应用或会话的精确资产版本和 digest。

它不能扫描全局资产库、根据任务类别推导授权或从已安装资产中自主选择。发现不
等于同意。

## 必须顺序

```text
明确选择或 Host 已批准附加
→ inspect
→ plan-load
→ 仅在 can_load_now=true 时 load
→ 消费 Runtime Capsule
→ 通过 Host 状态显示当前身份与作用域
```

适配器不得解包文件、读取原始 payload、绕过失败的 LoadPlan，或把签名当作内容
背书。

## 任务适用性

只有建立可选资产集合后才判断适用性。不适用边界会排除资产；存在实质不同取舍
的多份资产时让用户选择；没有明确匹配就跳过。

## 可见性

不要把受保护内部内容粘贴进普通回答，也不要隐藏加载事实。Host 应提供资产身份、
版本、作用域、profile、原因，以及停用、切换和回滚入口。

## 权威与失败

当前事实、用户意图、法律、安全政策和工具权限高于资产判断。校验、checksum、
签名、授权、兼容或解密失败会阻断该资产。被跳过或阻断的资产不得被表示为已加载。
